# Learning Source Library

This project keeps quest learning resources inline for backward compatibility. The new
source library is a generated artifact, not a live source of truth:

- Input: all links in `data/target-quests/*.json` where entry status is `available`.
- Output: `data/learning-sources.json` generated by `node scripts/build-learning-sources.mjs`.

## Generated Contract

`scripts/build-learning-sources.mjs` writes an object with:

- `version`: schema version.
- `generatedAt`: ISO timestamp.
- `summary`: target and source counts.
- `sources`: canonical source list sorted by URL.
- `conflicts.labelUrlConflict`: labels reused for multiple URLs.
- `conflicts.urlLabelConflict`: URLs used with multiple labels.
- `invalid`: collected malformed links.

## Why this exists

- Normalize duplicated source references without rewriting quest payloads yet.
- Give migration tooling stable IDs (`source-0001`, `source-0002`, ...).
- Provide explicit conflict visibility before a larger refactor to `sourceId` references.

Run:

```bash
node scripts/build-learning-sources.mjs --root . --output data/learning-sources.json
```

No UI path depends on this file yet.
